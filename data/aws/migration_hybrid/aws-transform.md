## AWS Transform

### Description
AWS Transform is an agentic AI service that accelerates enterprise modernization of legacy workloads — including full-stack Windows (.NET, SQL Server), mainframe (COBOL, PL/I), VMware, and custom code migrations — by deploying specialized AI agents to automate analysis, planning, dependency mapping, code refactoring, and transformation. Built on AWS's 19 years of migration experience, it can modernize Windows workloads up to 5x faster and VMware network configurations 80x faster than manual approaches. Transform supports natural language interaction and shared workspaces for cross-team collaboration. It is the recommended successor to AWS Migration Hub, AWS Application Discovery Service, and AWS Mainframe Modernization (managed runtime), all of which are no longer open to new customers. Pricing is based on consumption of transformation units.

### Use Cases
* Full-stack Windows modernization (e.g., automatically refactoring .NET Framework applications, SQL Server databases, and deployment pipelines to modern cloud-native equivalents — up to 70% operating cost reduction)
* Mainframe application modernization (e.g., using the mainframe agent to analyze COBOL/PL/I codebases, generate refactoring plans, and convert legacy batch programs from years to months)
* VMware workload migration to EC2 (e.g., automated discovery and dependency mapping of VMware VMs, network translation, wave planning, and EC2 instance right-sizing)
* Custom code and framework upgrades at scale (e.g., organization-wide Java version upgrades, Python runtime migrations, or API framework transitions across hundreds of repositories)
* Migration planning and portfolio discovery (e.g., automated assessment and dependency mapping of on-premises servers to plan migration waves — replacing Application Discovery Service)
* Cross-functional modernization collaboration (e.g., product, engineering, and infra teams tracking transformation progress in shared workspaces with real-time natural language queries)
