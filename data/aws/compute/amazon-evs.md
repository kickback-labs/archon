## Amazon Elastic VMware Service (EVS)

### Description
Amazon Elastic VMware Service (Amazon EVS) is the fastest path to migrate and operate VMware workloads on AWS. It runs VMware Cloud Foundation (VCF) software — including vSphere, vSAN, NSX, and vCenter — directly within an Amazon Virtual Private Cloud (Amazon VPC), allowing customers to retain their familiar VMware tooling, skills, and operational runbooks without re-platforming or re-factoring applications. Amazon EVS is an AWS-native service (not a third-party offering) that is generally available as of 2025, replacing the earlier VMware Cloud on AWS joint offering with Broadcom/VMware. Customers can choose to self-manage or engage AWS Partners for a fully managed experience. Amazon EVS seamlessly extends on-premises VMware environments to AWS, enabling data center exit strategies, disaster recovery, and cloud bursting. VCF subscription portability allows customers to bring existing VMware licenses or purchase new ones through AWS.

### Use Cases
* Data center exit and migration of VMware VMs to AWS without re-factoring or re-architecting applications
* Extending on-premises VMware capacity to AWS for cloud bursting during peak demand
* Disaster recovery (DR) for on-premises VMware workloads using AWS as the recovery site
* Running VMware-dependent third-party software (e.g., ISV applications certified only on VCF) on AWS
* Gradual modernization: run VCF workloads on EVS while selectively adopting AWS-native managed services (RDS, EKS, Lambda)
* Consolidating multiple aging VMware data centers into a single AWS-based VCF environment
